Small API.
Serious extensions.

Klack owns the lifecycle around your plugin, so you can focus on the behavior you actually want to add.

Autocomplete included

Import definePlugin from klack/sdk for typed definitions, setup context, button options, and cleanup contracts.

Zero restart workflow

TypeScript and JavaScript plugins are bundled in memory. Save a file and Klack applies the working set to every Slack window.

UI that stays mounted

Buttons and custom mounts are recreated when Slack’s React-managed DOM replaces a target or contribution.

Disable means gone

Styles, DOM watchers, mounts, listeners, timers, and registered cleanup callbacks are tracked in reverse order and removed automatically.

Extend Slack without weakening it.

Klack wraps Slack’s startup and prepends its generated preload while leaving sandbox: true and contextIsolation: true unchanged.

One file is enough.

Drop a TypeScript file into ~/.klack/plugins. Klack takes it from there.

~/.klack/plugins/hello.ts
import { definePlugin } from "klack/sdk"

export default definePlugin({
  name: "Hello",
  description: "Adds a friendly action",
  setup(klack) {
    klack.ui.addStyle(`
      [data-klack-button="Hello:wave"] {
        border-radius: 999px;
        padding: 6px 10px;
      }
    `)

    klack.ui.addButton({
      id: "wave",
      target: "[data-qa='message_input']",
      position: "before",
      label: "Wave 👋",
      onClick(_event, { button }) {
        button.textContent = "Waved!"
      },
    })
  },
})
